# Spec Me

Use when the user asks for a spec file before starting demo or prototype work.

## Workflow

1. Create a `SPEC.md` file at the current project root unless the user names another path.
2. Keep the spec focused on what must be true before implementation starts.
3. Include goals, scope, non-goals, key flows, data or API shape, acceptance criteria, risks, and open questions.
4. If the user is still exploring, ask the minimum number of questions needed to unblock the spec.
5. Do not start prototype code until the spec exists.

## Note shape

- objective
- constraints
- proposed solution
- acceptance criteria
- open questions

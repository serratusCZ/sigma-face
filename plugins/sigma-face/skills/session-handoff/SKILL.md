# Session Handoff

Use when the next session must continue the same thread.

## Workflow

1. Write a handoff note to `work/session-end/<timestamp>-handoff.md`.
2. Record exactly what was completed, what is blocked, and what should happen next.
3. Include any traps, decisions, and context that a fresh session would otherwise miss.
4. Update `work/session-end/LATEST.md`.

## Note shape

- current task
- completed work
- current blocker
- next steps
- never-do-again notes
- handoff advice
